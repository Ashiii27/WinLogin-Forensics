# Contributing to WinLogin Forensics

Contributions are welcome across parsers, anomaly detection logic, UI, and documentation.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/WinLogin-Forensics.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes and commit: `git commit -m "feat: describe your change"`
5. Push and open a Pull Request against `main`

## Commit Convention

Use conventional commits:
- `feat:` — new feature
- `fix:` — bug fix
- `docs:` — documentation only
- `test:` — adding or updating tests
- `refactor:` — code change with no feature or fix

## Areas Open for Contribution

- EVTX parser edge cases and extended Event ID support
- Registry hive parsing improvements
- Additional anomaly detection rules
- UI/UX improvements on Streamlit pages
- Sample data generation scripts
- Unit test coverage

## Reporting Bugs

Open an [Issue](https://github.com/Ashiii27/WinLogin-Forensics/issues) with:
- Steps to reproduce
- Expected vs actual behaviour
- Python version and OS
